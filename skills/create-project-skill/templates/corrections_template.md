# {{project_name}} Correction Layer

## Usage

This file is used to record incorrect conclusions, outdated content, or old knowledge that later needs to be overridden.

When making corrections, do not directly erase the old content. You must preserve:

- The old conclusion
- The new conclusion
- The reason for correction
- The effective version
- The impact scope

## Maintenance Rules

- A correction is not the same as deleting history
- Every correction should have source support
- If a correction affects the handoff workflow, `QUICKSTART.md` must also be updated
- If a correction affects the definition of success, `VALIDATION_RULES.md` must also be updated
